Web Components integration

This demo shows how you can use the brand new document.register method with jide.js to be able to create custom HTML elements for jide.js components.

This demo defines three new HTML tags:

  • jide-button
  • jide-accordion
  • jide-titledpane

These elements can be used just like any other HTML element. As soon as the controls are registered, the elements are updated to behave exactly like you'd expect from a jide.js control.

Since jide.js doesn't support Web Components natively (the spec is not supported by most browsers and needs to be shimmed) you need to use a function like the one provided by the register module in this demo. You also need to use a shim like Mozilla Web Components if you'd like to support most browsers.

For this demo, the element doesn't support any of the jide.js components properties. Instead, you need to access its component property for access to the actual component.
  • You can use custom elements, prefixed with jide- in your HTML that are transformed to components upon startup.
  • You can use document.createElement to directly create such components.
  • You can use setAttribute to set a property on the actual component.
  • Children of the custom element are placed under the components control when possible.
